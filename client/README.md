# Quiz Master -בתיה שאהן פרויקט גמר 
# React + Node.js

מערכת מלאה לניהול וביצוע חידונים אינטראקטיבית הכוללת ממשק ניהול מתקדם, אבטחת מידע ועיצוב רספונסיבי מלא. הפרויקט מציג יישום מקצה לקצה של צד לקוח (React) וצד שרת (Node.js), תוך עבודה מול מסד נתונים MongoDB Atlas.

##  תכונות מרכזיות שמומשו בפרויקט

* **ניהול משתמשים והרשאות:**
 .הפרדה מלאה בין אורחים, משתמשים רשומים ומנהלי מערכת
  .(JWT Authentication) תצוגת האתר והפעולות הזמינות משתנות דינמית לפי הרשאת המשתמש 
* **פעולות CRUD מלאות:** 
:מערכת לניהול חידונים 
(יצירה, הצגה, עריכה ומחיקה),
:ניהול משתמשים
 הצגה, עדכון מעמד המשתמש 
 (ומחיקת משתמשים מורשית למנהל הראשי בלבד)
ניהול תוצאות: הצגה מיון ומחיקה

* **אבטחה** 

 הצפנת סיסמאות בשרת באמצעות `bcryptjs` 
 ושימוש ב-Mongoose Middleware (`pre-save`).
    * הגנה על נתיבי API (Routes) 
    באמצעות Middleware ייעודי לבדיקת טוקנים והרשאות.

* **אימות נתונים (Validation):**
 בדיקות תקינות קפדניות בצד השרת באמצעות `Joi`, ושימוש ב-`react-hook-form` בצד הלקוח.

* **מדיה וקבצים:** 
מערכת להעלאת תמונות לחידונים ולשאלות המבוססת על `Multer`.

* **חסימת שבת (Custom Middleware):**
 פיתוח רכיב צד-שרת החוסם את הגישה לאתר משישי אחר חצות עד מוצאי שבת!! (מחזיר סטטוס 503).

* **חווית משתמש ועיצוב:** 
ממשק מודרני מבוסס `MUI` עם תמיכה מלאה ב-RTL (עברית),
 אנימציות רקע ורספונסיביות מלאה לכל סוגי המסכים.

##  טכנולוגיות בשימוש 

### צד שרת 

* **Node.js & Express** - תשתית השרת וניהול נתיבים.
* **MongoDB Atlas & Mongoose** - מסד נתונים וניהול סכמות.
* **Joi** - אימות נתונים (Validation).
* **JsonWebToken (JWT) & Bcryptjs** - אבטחה, אימות וניהול הרשאות.
* **Multer** - טיפול בהעלאת קבצי תמונות.
* **Cors** - ניהול גישה בין שרתים.

### צד לקוח (Frontend)
* **React (Vite)** - תשתית הפיתוח.
* **Redux Toolkit & React-Redux** - ניהול State גלובלי.
* **MUI (Material UI) & Emotion** - ספריית עיצוב ורכיבי ממשק.
* **Axios** - תקשורת מול השרת (כולל Interceptors).
* **React Router Dom** - ניהול ניווט ודפים.
* **React Hook Form** - ניהול טפסים חכם.
* **Framer Motion** - אנימציות ממשק.
* **React tsparticles & Confetti** - אפקטים ויזואליים של רקע והצלחה.

##  ארכיטקטורה
הפרויקט בנוי במבנה מודולרי המפריד בין Models, Controllers ו-Routes לשמירה על סדר ותחזוקתיות.
 מסד הנתונים מבוסס על קשרי גומלין בין אוספי משתמשים, חידונים ותוצאות.

##  הערות מיוחדות
* **ייחודיות המנהל:** 
המערכת מזהה את המנהל הראשי במערכת ולא מאפשרת לשנות את תפקידו או להימחק
 .כדי למנוע מצב של מערכת ללא ניהול

* **מנהל על (Super Admin):**
 קיים מנגנון הרשאות מיוחד למנהל-על לניהול מנהלים אחרים ומחיקת ציונים מההיסטוריה.

## שמירת ציונים חכמה 
.(sessionStorage)המערכת מאפשרת לכולם לשחק כאורחים ולשמור את תוצאת המשחק שלהם בזיכרון הזמני 
 אם האורח מחליט להירשם או להתחבר מיד לאחר סיום החידון,
  המערכת מזהה את הציון ה"ממתין" ומשייכת אותו אוטומטית לחשבון המשתמש שלו במסד הנתונים.


##  הוראות הפעלה (הרצה מקומית)

### 1. הגדרת שרת (Server)
 היכנסו לתיקיית `server` והריצו `npm install`.
 הריצו את השרת: `npm start`.

### 2. הגדרת לקוח (Client)

 היכנסו לתיקיית `client` והריצו `npm install`.
 ודאו שבקובץ `.env` קיים:
 `VITE_API_URL=http://localhost:3001/api`
 הריצו את האפליקציה: 
 `npm run dev`.
